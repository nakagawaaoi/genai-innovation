# このプロジェクトのルール（AIエージェント共通）

このリポジトリは「生成AIイノベーション合同会社」のコーポレートサイト。`index.html` / `style.css` / `main.js` が本番サイトの実体（GitHub Pages公開を想定、`.nojekyll`配置済み）。
**Antigravity と Claude Code の両方から編集される。** 姉妹リポジトリ（個人LP）と同じ運用ルールをここにも適用する。

## 作業ルール

- **作業を始める前に `git pull` で最新を取り込む。** 普段はMac 1台・1フォルダを2つのAIツールで交互に使うだけなのでコンフリクトのリスクは低いが、スマホ／別PC／GitHub上で直接編集した後にMacで作業を再開する時は必ず1回 `pull` する。
- 同じファイルを両ツールで**同時に**編集しない（交互運用を守る）。
- コミットメッセージの目印（どちらのツールが変更したか履歴で一目で分かるようにする）：
  - Antigravity → 先頭に `[Antigravity]` を付ける
  - Claude Code → `Auto-commit via Claude Code` の名乗りを含める

## 自動コミット & プッシュ（ユーザー承認済み・恒久指示）

**ファイルを変更したら、そのタスクの区切りごとに確認を挟まず自動で以下を行う。**

1. `git add` で変更点をステージ
2. `Auto-commit via Claude Code: <変更内容が分かる簡潔な説明>` の形式でコミット
3. `git push` で `origin/main` に即座にプッシュ

- ここは個人運営のシンプルな静的サイトリポジトリで、PRフローは使わず`main`へ直接コミットする運用（既存の履歴と同じ、チーム開発ではないので許容）。
- 通常Claude Codeは`git push`のような影響の大きい操作の前にユーザー確認を取るが、このプロジェクトに限っては本ファイルの記載をもって事前承認済みとする。毎回「プッシュしていいですか」とは聞かない。
- ただし `git push --force`、`git reset --hard`、ブランチ削除などの破壊的操作は対象外。これらは引き続き必ず確認を取る。
- コミット前に `git status` / `git diff` で意図しない変更（他ツールの未確認の作業途中ファイルなど）が混ざっていないかは確認してよいが、確認のために作業を止めない。

## セキュリティ

- APIキー・パスワード・トークンの類は絶対にコミットしない。使うなら `.env` に置き、`.gitignore` で除外する（このリポジトリは現状Vanilla HTML/CSS/JSで秘密情報を扱う機能は無いが、今後フォーム送信やAPI連携を追加する場合は要注意）。
- 一度 `push` したものはGitHubの履歴に残り続けると心得る（削除・非公開化しても完全には消えない）。
- 会社概要ページの住所等、公開してよい情報かどうかは事業上の判断として別途確認する（合同会社の登記情報として開示が前提なら問題なし）。

## 注意（Google Driveとの同居）

- このフォルダは Google Drive の同期下にある。Gitの`.git`管理とDriveの常時同期は相性が悪く、同期中に激しく編集すると`.git`が破損する事故が起こり得る。
- 対策：編集はDriveの同期が落ち着いているタイミングで行う。`push`できていればGitHub側に本体のバックアップがあるので、Drive自体をバックアップ目的で過信しない。
- 恒久対策としては、Drive外のローカルフォルダ（例 `~/projects/`）への移設が望ましいが、移設するとAntigravity側が記憶しているフォルダパスの再設定が必要になるため、実施するかどうかはユーザー判断で決める（このファイルの指示だけでは移設を実行しない）。

## 知っておくと得

- 壊れたら戻せる：`git log` で履歴を見て、`git revert <commit>` で特定の変更だけを打ち消せる。
- `main`への直接push運用は、個人／小規模運営のサイトである前提で許容している（チーム開発になったらPRフローへの切り替えを検討）。
