# 運用ルール

このプロジェクトはAntigravityとClaude Codeの両方から編集される。GitHub上の状態を常に最新に保つため、以下を徹底すること。

## 自動コミット & プッシュ（ユーザー承認済み・恒久指示）

**ファイルを変更したら、そのタスクの区切りごとに確認を挟まず自動で以下を行う。**

1. `git add` で変更点をステージ
2. 変更内容が分かる簡潔なコミットメッセージでコミット
3. `git push` で `origin/main` に即座にプッシュ

- ここは個人運営のシンプルな静的サイトリポジトリで、PRフローは使わず`main`へ直接コミットする運用（既存の履歴と同じ）。
- 通常Claude Codeは`git push`のような影響の大きい操作の前にユーザー確認を取るが、このプロジェクトに限っては本ファイルの記載をもって事前承認済みとする。毎回「プッシュしていいですか」とは聞かない。
- ただし `git push --force`、`git reset --hard`、ブランチ削除などの破壊的操作は対象外。これらは引き続き必ず確認を取る。
- コミット前に `git status` / `git diff` で意図しない変更（他ツールの未確認の作業途中ファイルなど）が混ざっていないかは確認してよいが、確認のために作業を止めない。
